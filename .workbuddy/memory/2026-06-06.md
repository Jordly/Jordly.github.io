## 18:20 - 筛选栏v4未生效问题排查与修复

### 问题
用户更新后仍然看到旧版药丸按钮组筛选栏，v4下拉菜单未生效。

### 根因分析
filterBarNew.js 虽然在 app.js 之后加载，但运行时覆盖 `renderFilterBar()` 没有生效。可能原因：
1. 浏览器缓存了旧版 app.js（虽然版本号已更新）
2. Cloudflare Pages 边缘缓存延迟
3. JS 执行顺序/作用域问题导致函数覆盖失败

### 解决方案
**最可靠的修复：直接把 v4 筛选栏实现内联到 app.js 中**，不再依赖 filterBarNew.js 的运行时覆盖。

### 修改内容
1. **app.js filterState 结构**：`brand`/`category` 从字符串改为数组，`platforms` 新增数组，删除 `moreExpanded`
2. **app.js renderFilterBar()**：完全替换为 v4 版本（两行下拉菜单 + 搜索多选面板）
3. **app.js 辅助函数**：添加 `onFilterTimeChange`, `applyTimeFilter`, `toggleMultiFilter`, `toggleFbSearch`, `renderFbOptions`, `onFbOptionClick`, `applyFbMulti`, `clearFbMulti`
4. **app.js resetFilters()**：适配数组类型的 brand/category/platforms
5. **app.js getFilteredProjects()**：brand/category 从字符串比较改为数组包含检查，新增 platforms 筛选
6. **index.html**：删除 filterBarNew.js 引用，更新所有 CSS/JS 版本号到 `20260606182047`

### 教训
运行时函数覆盖不可靠。对于核心 UI 组件，应直接内联到主文件中。

2. **CSS !important 会阻止JS控制**：需用class切换代替
