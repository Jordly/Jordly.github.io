### 问题2：弹窗无响应（10:25 - 11:45）
**用户反馈**：
1. 点击"＋"按钮没反应
2. 点击"复制权限"没反应
3. 导出配置是浏览器原生下载弹窗

**排查过程**（走了弯路）：
1. 第一次：以为是ID冲突 → 改成唯一ID方案（错误）
2. 第二次：以为是querySelector问题 → 改成querySelector（错误）
3. 第三次：用户提供了关键线索 → 找到真正原因（正确）

**关键线索**（用户提供的）：
> "我鼠标放在+按钮上不点击的状态，是小手；点击后，是鼠标箭头，不能再次点击，且没有任何弹窗画面显示"

**真正的根因**：
**CSS类名不匹配**！
- CSS中定义：`.sd-confirm-show { opacity: 1; }`
- 但 `showPromptModal()` 中添加：`overlay.classList.add('sd-prompt-show')`
- 结果：弹窗创建了但 `opacity: 0`（看不见）

**修复方案**：
把 `showPromptModal()` 和 `showSelectModal()` 中的所有 `sd-prompt-show` 改成 `sd-confirm-show`

**Git提交**：
- commit 9ca716b: "修复弹窗不显示：CSS类名sd-prompt-show→sd-confirm-show (v202607031145)"

## 教训（重要！）

### 1. 用户线索非常关键
"点击后变成箭头，且不能再次点击" → 说明点击事件被触发了，但弹窗创建了却看不见

### 2. 不要盲目改代码
应该先检查：
1. 控制台有没有报错
2. CSS类名是否匹配
3. DOM元素是否被正确创建

### 3. CSS类名要统一
所有弹窗都用同一个CSS类名（`sd-confirm-show`）来控制显示/隐藏
