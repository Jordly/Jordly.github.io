## 项目对比弹窗乱码修复

### 问题
用户选中两个项目点击"开始对比"后，弹窗显示区域出现乱码（如 `鍒?/span>`、`缁村害`、`宸㈡紓`）。

### 根因
这些中文字符在 Git 历史中曾被错误保存为 GBK/GB2312 编码（中文被当 UTF-8 解析），典型 mojibake：
- `鍒?` 应为 `分`（综合得分单位）
- `缁村害` 应为 `维度`（表头）
- `宸㈡紓` 应为 `差距`（表头）
- `鎵归噺缂栬緫鍔熻兘寮€鍙戜腑` 应为 `批量编辑功能开发中`
- `鏃犲巻鍙叉暟鎹?` 应为 `无历史数据时返回平直线段`（注释）

### 修复
在 `app.js` 中逐一替换（涉及两处对比模态函数 + 其他位置），全部改为正确 UTF-8 中文。

**Git提交**：`cdcdd03`

## 核心知识能量池版块优化（方案A+B+C+D）

### 用户提出的4个问题
1. 卡片/标签点击无效（原 showKnowledgeDetail 只是 alert）
2. 卡片文字太多太乱，不便于快速定位
3. 内容应定位"管理者通用技能"，不是一线基础客服内容
4. 必须与系统数据管理"知识条目表"打通同源

### 改动清单
- **方案A（交互）**：`showKnowledgeDetail()` 改为自定义弹窗（含完整描述、标签、编辑/删除）；标签点击 `kypFilterByTag()` 实现筛选；添加/编辑/删除按钮接入 `addKnowledge/editKnowledge/deleteKnowledge/showKnowledgeForm/submitKnowledgeForm`，写回 KNOWLEDGE + saveKnowledge()。
- **方案B（UI精简）**：卡片去掉 scope/perm badge 与底部 tag 群；描述改为1行截断（`.kyp-card-short`，用新增 short 字段）；卡片高度明显缩小。
- **方案C（内容升级）**：`DEFAULT_KNOWLEDGE` 8条默认数据改为管理视角（人力成本优化、AI工具落地、经验复制SOP、离职应急、升级投诉沟通、新任主管上手、BPO费效比、智能质检），新增 `category`（管理方向：团队管理/成本控制/效率提升/风险防控/体系搭建/沟通协作）与 `short` 字段。
- **方案D（数据打通）**：
  - **修复 `initKnowledge()` bug**（if块为空，每次刷新用默认覆盖用户数据）→ 改为带种子版本号 `KNOWLEDGE_SEED_VERSION='2026-07-manager'`，本地有同版本数据才加载，否则重新播种。
  - 知识条目表 `SYSTEM_DATA_TABLES.knowledge.fields` 扩展为 type/category/short/description/tags/scope/permission 等，type 选项与能量池统一（6类）。
  - colDefs 增加"管理方向/浏览/下载"列，加 `goEnergyPool:true`；详情页加"📖 在能量池查看"按钮跳转 renderModule('knowledge')。
  - 两侧共用 `KNOWLEDGE` 数组引用，双向实时同步。

### 注意
- 因 initKnowledge 历史 bug，用户此前无法持久保存知识数据（刷新即被默认覆盖）。修复后旧默认数据会被新管理视角种子替换——符合用户"内容升级"诉求。
