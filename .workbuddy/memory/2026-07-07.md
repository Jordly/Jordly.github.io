- 图例样式仅 8px 小方块 + 2px 圆角 + 12px 字，粗糙。

- 把图例从底部独立区块**移入筛选栏最右侧**，与"全部/分类"标签同处一行，间距自然不突兀。

## 核心知识能量池 · 后续优化续（防填充/删图例/拖拽排序）

### 1. 搜索框被浏览器自动填充为登录账号（v202607071415）
- 根因：浏览器把知识搜索框误识别为"用户名"字段，自动填入 admin。
- 修复：搜索框加 `autocomplete="off"` 初版无效（Chrome 忽略），后续强化为 `autocomplete="nope"` + 改名 id/name（`kyp-search-field`/`kyp_search_query`）+ `type="search"` + `spellcheck="false"`（v202607071455 一并处理）。

### 2. 删除筛选栏图例（v202607071420）
- 用户认为图例无意义，删掉筛选栏右侧 SOP/风控/成本… 图例块。删图例时不慎把搜索框的 `autocomplete="off"` 一并误删（恢复成可自动填充），在 v202607071455 的强化防填充中已重新修正。

### 3. 卡片拖拽排序（v202607071455 初版 + v202607071505 修复）
- 初版实现：卡片 `draggable="true"` + `ondragstart/ondragend`，grid `ondragover/ondrop`；`kypDragStart/kypDragOver/kypDragEnd/kypDrop` 全局函数；拖拽后 splice 重排 KNOWLEDGE + saveKnowledge + renderModule('knowledge')。
- **bug（v202607071505 修复）**：`kypDragStart` 用 `e.target.dataset.id`，从卡片内文字/子元素拖动时 `e.target` 是子元素（无 data-id）→ `parseInt(undefined)=NaN` → drop 里 `if(!kypDragSrcId) return` 因 `!NaN===true` 永远跳过排序，松手无变化。
- **修复**：dragstart 改用 `e.currentTarget`（绑定在卡片上，必带 data-id）；drop 兜底取 `.kyp-drag-over` 高亮卡片，避免松手在缝隙落空。
- **教训**：HTML5 拖拽事件中取源元素 id 必须用 `e.currentTarget` 而非 `e.target`。

### 涉及版本/提交
- v202607071415 `...` 防填充初版
- v202607071420 删图例
- v202607071455 防填充强化 + 拖拽初版
- v202607071505 `f9b2122` 拖拽修复（仅改 currentTarget，用户反馈仍无效）

### 4. 搜索框防填充最终方案 + 拖拽重做（v202607071530 `da144a0`）
- **搜索框**：`autocomplete="nope"` 在 Chrome 下仍被忽略。最终改为 `autocomplete="off"` + `readonly` + `onfocus="this.removeAttribute('readonly')"`——浏览器不填充只读输入框，用户点击聚焦时才可输入。这是可靠的防填充方案。
- **拖拽**：放弃"drop 后 splice 数组 + renderModule 重渲染"的链路（中间环节任一异常都会导致顺序不变）。改为 drop 时用 DOM `insertBefore` **直接移动卡片节点**（所见即所得），再由 DOM 顺序反推 `KNOWLEDGE` 数组顺序 `KNOWLEDGE.sort(...)`，最后 `saveKnowledge()`。视觉反馈改用 `.kyp-dragging`（半透明）/`.kyp-drag-over`（蓝色高亮）类名，不再用 inline opacity。
- **教训**：HTML5 拖拽排序最稳妥做法是"drop 时直接 DOM 移动 + 反推数组"，不要依赖整页 JS 重渲染来体现顺序变化。

### 5. 拖拽仍无效 → 改用鼠标事件实现（v202607071545 `46d1261`）
- 用户两次反馈 HTML5 `draggable` 拖拽无效（松手顺序不变）。根因未 100% 确认（疑似 drop 未触发 / Chrome 对带 onclick 的 draggable 卡片拖拽行为异常），继续调试 HTML5 拖拽性价比低。
- **最终方案**：彻底弃用 HTML5 `draggable`，改用原生鼠标事件手写拖拽：
  - `kypCardMouseDown(e,id)`：记录起点 + 绑定 document 级 `mousemove`/`mouseup`
  - `kypCardMouseMove`：移动超 5px 判定为拖拽，源卡加 `.kyp-dragging`（半透明），`elementFromPoint` 找悬停目标卡加 `.kyp-drag-over`
  - `kypCardMouseUp`：按鼠标 Y 中线决定插到目标卡前/后，`insertBefore` 移动 DOM，反推 KNOWLEDGE 数组 `sort` + `saveKnowledge`
  - `kypCardClick(e,id)`：若 moved 则拦截（不弹详情），否则 `showKnowledgeDetail`
- CSS：`.kyp-card { user-select:none }` 防拖拽误选文字；去掉 `draggable` 的 cursor 规则。
- **关键经验**：当 HTML5 drag&drop 在真实环境不稳定时，手写 mouse 事件拖拽是更可靠、跨浏览器一致的替代方案。

### 6. 卡片拖拽升级为 iOS 长按重排（v202607071605 `7d392d8`）
- 用户要求：类似手机端长按 APP 图标拖拽——**长按进入拖拽**，拖动时**其他卡片实时让位**，松手落位后其他卡片自动排序。
- 实现：
  - 长按判定：`kypCardMouseDown` 启动 450ms 计时器；计时前移动 >8px 则取消（视为普通点击）；计时器触发才进入拖拽。
  - 进入拖拽（`kypStartDrag`）：被拖卡 `position:fixed` 跟随光标（class `.kyp-drag-lift`：放大1.05+微旋转+阴影），原位插入 `.kyp-drag-placeholder`（虚线框占位）。
  - 拖动（`kypMoveDrag`）：用 FLIP 动画（`kypFlip`）实时把占位符移动到目标位置，其他卡片平滑让位（transform translate + transition 0.2s）。
  - 松手（`kypFinishDrag`）：卡片落到占位符处，按 DOM 顺序反推 `KNOWLEDGE.sort` + `saveKnowledge`；`suppressClick` 阻止误触发详情。
- CSS：`.kyp-card.kyp-drag-lift`（放大旋转阴影）、`.kyp-drag-placeholder`（虚线蓝框）、`.kyp-card { user-select:none }`。
- **关键经验**：iOS 风格实时重排 = 浮动卡(fixed) + 占位符 + FLIP 过渡；长按计时器需与"移动即取消"配合，避免与点击打开冲突。

### 核心 Bug 修复（致命）

### 关键经验
- 搜索框 oninput 若重渲染整页会丢焦点 → 用全局 `handoverFilter` 状态 + 局部刷新列表容器 `#handover-list-wrap`（只换 tbody 区域），搜索框本身不重渲染。
- Edit 插入大段代码时，**old_string 不要包含含字面 `\n`(反斜杠n) 的 alert 行**，否则 JSON 转义使 `\n` 变真实换行导致不匹配；改用函数开头两行作锚点。
- 插入新函数到某函数前时，若 old_string 以目标函数声明开头、new_string 又以其开头，会**重复函数声明导致花括号不闭合**（本次踩坑：node --check 报 "Unexpected end of input"）。插入块应直接以注释开头，末尾接回原函数前两行即可。
- 旧交接数据无新字段（type/checklist 等）→ 渲染处一律用 `||` 兜底（如 `h.type||'—'`、`h.checklist||[]`），保证兼容不报错。
