## 登录记住我功能优化（方案B）

### 问题描述
用户周一登录系统，发现不需要输入密码就自动登录了（周六日无任何操作），记住密码的有效期太长。

### 根本原因
1. "记住我"复选框默认是**勾选状态**（`checked`）
2. 勾选后有效期是 **7天**（604800000毫秒），周末2天当然还在有效期内
3. 提示文字写的是"1小时内免登录"，和实际7天不一致

### 修改内容（方案B）
1. **默认不勾选**：移除 `<input ... checked>`，用户需要主动勾选才会记住
2. **有效期7天→1天**：`604800000` → `86400000`（24小时）
3. **修正提示文字**："1小时内免登录" → "1天内免登录"
4. **涉及文件**：`index.html`（3处）+ `app.js`（1处）

## 重要教训：login.html 和 index.html 是两套登录UI

### 问题
用户反馈修改后页面没变化，清缓存也没用。原因是：
- **`index.html`**：系统主页面，**内部有一个登录弹窗**（modal），改的是这里
- **`login.html`**：**独立的登录页面**，未登录时系统会跳转到这个页面，之前完全没改

### 根因
系统有两个入口：
1. 访问 `chanseen.pages.dev` → 已登录直接进系统，未登录跳转到 `login.html`
2. `login.html` 是**完全独立的HTML文件**，有自己的checkbox和JS逻辑

### 修复
同时修改了两个文件：
- `index.html`：登录弹窗的"记住我"（之前已改）
- `login.html`：独立登录页面的"记住我"（本次补充修改）

### 经验总结（重要！）
**以后改登录相关功能，必须同时检查并修改两个文件**：
- `index.html` 中的登录弹窗（modal）
- `login.html` 独立登录页面

  - 删除 `Math.random`，改用 PROJECTS 真实 profitRate；实际成本列显示成本预算（选项A：预算即实际）
  - 新增 导出/录入/刷新 按钮 + 每行"编辑"按钮
  - `editCostRecord(pid)` / `addCostRecord()` / `exportCost()`：编辑写回 PROJECTS → `saveProjects()` → `CloudBaseSync.saveAll()` 云端同步
  - 中台 projects 表 `fields` + `colDefs` 补上 营收(万)/成本预算(万)/利润率(%) 3列（双入口一致）
- **新增通用样式**：`.cost-actions`/`.cost-table-header`/`.cost-add-btn`/`.btn-xs`/`.profit-table-wrap`/`.profit-table-v2`

## 成本与利润管理 - 三项UI微调（用户反馈）

### 1. 删除重复的"录入数据"按钮
`renderCost()` 里有两个完全一样的"录入数据"按钮（顶部标题区 + 表格卡片标题区），都调用 `addCostRecord()`。已删除表格卡片区的那个，只保留顶部标题区，和其他模块设计一致。

### 2. 编辑按钮淡化
每行"✏️ 编辑"原本用 `btn-xs btn-primary`（蓝色实心大按钮），太抢眼。改为轻量文字链接 `.cost-edit-link`：灰色文字，hover才变蓝+下划线背景，不喧宾夺主。

### 3. 表头横线修复
根因：`.data-table th` 基础样式有 `border-bottom: 2px rgba(24,95,165,0.15)`，叠加到 `.profit-table-v2 thead th` 的 `border-bottom: 2px #e2e8f0`（后又被 `::after` 极光条覆盖），多层border叠加导致表头文字像被横线穿过。修复：利润表 `border-bottom` 改为 `none`，只保留极光渐变 `::after` 装饰线。

### 关于删除按钮
用户问是否加删除功能，建议**不加**——成本页数据来自中台 `PROJECTS`，删除=删整个项目档案（风险大），删除应在系统数据管理（中台）操作。
