## 🔒 权限管理系统（2026-07-02 优化）

### 角色系统
- ROLES改为动态系统，从localStorage加载（key: `chansee_roles`）
- 内置角色6个：超级管理员、管理员、客服总监、客服经理、客服主管、项目伙伴（不可删除）
- 支持自定义角色：新增、编辑、删除

### 权限配置
- 权限数据存储在localStorage（key: `chansee_permissions`）
- 每个角色对每个模块有7个操作权限：visible（可见）、view（查看）、edit（编辑）、import（导入）、export（导出）、manage（管理）、scope（数据范围）
- 不支持的操作会完全隐藏（不是禁用）

### 前端权限拦截
- 登录成功后自动调用`filterNavByPermissions()`过滤导航菜单
- `renderModule()`函数开头检查权限，无权限则跳转第一个有权限的模块
- 辅助函数：`hasPermission(module, action)`、`canViewModule(module)`

### 角色管理功能
- 新增角色：点击"＋"按钮，输入角色名称
- 编辑角色：hover角色项，点击"✏️"（仅自定义角色）
- 删除角色：hover角色项，点击"🗑️"（仅自定义角色），显示受影响用户
- 复制权限：点击"📋从其他角色复制权限"按钮

### 受影响用户提示
- 修改角色权限时，显示"此修改将影响X个用户"
- 删除角色时，提示用户角色将被设置为"项目伙伴"
